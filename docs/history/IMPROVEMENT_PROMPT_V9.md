# Improvement brief V9 — expand the scope: from a lemma reconstructor to a full lexicographic engine

You are an expert in comparative Slavic linguistics, Rust, computational
lexicography, and evaluation methodology. Eight briefs took the engine from
27.5% to **41.65% exact / 49.59% normalized top-1**, and V8 added a
statistics-grade harness (seeded dev/holdout split, paired significance,
bootstrap CIs, validated isotonic calibration). The audit is now unambiguous:
~77% of the remaining benchmark error is editorial word-choice or an evidence
gap that no answer-blind rule can close, and the honest recoverable headroom is
+2–3pp per stage (see `target/eval/oracle-ladder.md`).

V9 is therefore different in kind: **stop optimizing one number and widen what
the engine can do.** The benchmark stays as a regression floor; the new work is
scope. Every expansion must arrive with its own leakage-free benchmark, built
BEFORE the feature — the V8 lesson is that categorical evidence and a
pre-registered metric beat cleverness every time.

## 1. Current state (verify these numbers first — they must reproduce)

- Pipeline (`evaluate`): **41.65% exact / 49.59% normalized** top-1, 16,300
  meanings; holdout gap +0.96pp; CI floor 39.5%.
- Site path (`corpus-eval`): 58.6% exact / 63.1% normalized on 7,358 scorable.
- Proto engine (`proto-eval`): 20.1% coverage, 46.68% exact on linked.
- Calibration (`target/eval/methodology.md`): raw score ECE 0.185 (overconfident);
  dev-fit isotonic calibrator reaches **holdout ECE 0.013** — measured but NOT
  yet wired into any product surface.
- Site: 21,404 generated pages + 12,269 official-only pages; 4,793 official
  lemmas reproduced; 2,066 novel words in `data/novel-words.tsv`; 216 blank
  inflection cells; 1,657 multi-word official lemmas render as official-only
  pages (never generated).
- 68 tests green; ladder ends at `ConsensusConfig::production()`.

## 2. Rules of engagement (unchanged core + V8 instruments now mandatory)

1. **Measure everything.** Consensus rules stay behind `ConsensusConfig` flags
   on ladder rungs. Every NEW capability ships with its own benchmark command
   and a committed report under `target/eval/` — no feature lands without a
   number attached. Regression test per fix; revert honestly and record it.
2. **No leakage** — the generator never sees `isv` (existing paths) and a new
   benchmark's gold side must never feed its generation side.
3. **Holdout discipline** — develop against the dev split, report both splits;
   a rule that gains on dev and not on holdout does not ship
   (`methodology.md` is regenerated by `evaluate`; the split hash must never
   change).
4. **Significance gate** — a kept rule needs the paired sign test on its keep
   metric, not just a positive delta; p ≳ 0.05 is "provisional", documented as
   such.
5. **Deterministic static generation** — no server, no DB, no network at build
   time; everything reproducible byte-for-byte from committed caches.

## 3. The expansion tracks, ranked

### Track A — Productive derivation: generate word FAMILIES, not lemmas (highest value)

The engine reconstructs one citation form per cognate set. Interslavic word
formation is regular and documented (RULE_SPEC §3, [DERIV], steen derivation
tables) — and V8 proved suffix behavior is categorical in the official lexicon
(59:0, 516:0, 152:0). Build a **derivation layer** that, from each reconstructed
stem, generates the family: abstract `-osť`, agentive `-telj`/`-teljka`/`-nik`,
adjectives `-ny`/`-livy`/`-sky`, adverb `-o/-ě`, verbal noun `-ńje`, diminutives,
aspect pairs (`-ovati/-yvati`, prefix perfectivization), negations, and the
morphophonemic alternations at each seam (first palatalization before `-ec/-ica/
-ina/-ok/-sky/-stvo`, fill vowels, O⇒E after softs).

- **Benchmark first** (`derive-eval`): the official dictionary contains
  thousands of derivationally related lemma pairs (найди их by shared root +
  POS pattern). Hold the base lemma, generate the derivative, score
  exact/normalized — the same leakage-free framing as `evaluate`.
- Site impact: derived-family blocks on every entry page, cross-linked; novel
  derivatives beyond the dictionary flow into Track C with calibrated
  probabilities.
- The root-consistency invariant is the test oracle: a family whose members
  disagree on the root spelling is a bug by definition.

### Track B — Multi-word lemmas, reflexives done fully, and verb aspect

1,657 official lemmas are multi-word (`a takože`, `adamovo jablȯko`) and today
only render as official-only pages. Quantify what is generatable: adjective+noun
collocations where both words are already reconstructed; verb + `sę`; phrasal
adverbs. Extend `evaluate` with a multi-word slice (currently filtered out by
`is_benchmarkable`) so the denominators are honest. Separately: aspect-pair
detection (the dictionary marks aspect; the cognates attest both stems) — cite
both members, link them, and benchmark pair accuracy.

### Track C — The novel-vocabulary pipeline (calibration finally earns its keep)

`data/novel-words.tsv` (2,066 words) is a flat file. Make it a **curated
proposal pipeline**:

- Wire the **isotonic-calibrated probability** (dev-fit, holdout-validated —
  the recipe is in `eval.rs::write_methodology`) into the corpus path so every
  novel word carries *P(would match an official decision)*, not the raw
  overconfident score.
- Rank, bucket (propose / review / discard), and render a "Predloženja" section
  on the site with the full evidence trace per word; `data/curation-notes.example.json`
  is the intended human-review format — implement its ingestion
  (`overrides.rs` is the pattern).
- Benchmark the FILTER, not just the words: on the official benchmark, what
  precision/recall does each probability threshold achieve? That same threshold
  then justifies the novel-word buckets.

### Track D — Flavorization and script layer

The site displays one etymological spelling. Interslavic officially has
standard/etymological levels plus national flavors, and the mapping is already
half-built (`orthography.rs::to_standard` one way; `russian_translit.rs` for
display). Build the **flavorizer**: etymological → standard → Cyrillic →
national flavors (Northern `-ť`, West `ȯ→e`, South vocalizations), as a pure
function with round-trip property tests (standard(flavor(w)) == standard(w)).
Every entry page gets a spelling table; the search index gains flavored keys
(a user typing `medžu`, `меджу` or `miedzu` should land on the same page).

### Track E — Evidence growth (only where the audit says it pays)

`0-root-absent` is 21.8% of misses — an extraction ceiling, not a rule ceiling.
Before touching rules: quantify how much of it is recoverable evidence. Sweep
the dump for cognates cited under different lemma shapes (perfective/imperfective
mismatches, orthographic variants, dialect labels currently dropped), and add
the four missing native-Wiktionary editions (uk/sr/bg/sl) to `extract-enrich`.
Report the root-absent rate before/after — that number, not the headline, is
this track's metric. The remaining oracle levers (proto-link +2.7pp,
representative +2.3pp) stay open as a small accuracy lane if an idea survives
the significance gate.

### Track F — Inflection completeness and grammar validation

216 inflection cells are blank and the tables are unvalidated. Diff the
generated paradigms against the `interslavic` crate's grammar for every page;
add a `inflect-eval` that counts cell-level agreement, fills the known gaps
(athematic neuters, i-stem instrumentals, irregular plurals from RULE_SPEC
§3.1), and fails CI on paradigm regressions.

### Track G — Distribution artifacts (cheap, high leverage)

From the same build: a Hunspell dictionary (`.dic`/`.aff` from the inflection
engine), a frequency-free Anki deck (word / gloss / evidence / etymology), and
a stable machine-readable JSON export of the whole lexicon (the site's data
without the HTML). Each is a deterministic `export` subcommand writing under
`site/downloads/`, listed on the site, license-stamped CC BY-SA.

## 4. Suggested order of attack

1. Reproduce §1. Build the **derive-eval benchmark** (Track A) before any
   derivation code — its baseline number on the current engine is itself a
   deliverable.
2. Track A derivation layer against it; ship families to the site.
3. Track C pipeline (the calibrator is sitting there validated and unused).
4. Track D flavorizer (pure function + property tests, low risk, big UX win).
5. Tracks B/E/F as the numbers dictate; Track G whenever a build artifact
   stabilizes.

## 5. Honesty requirements (unchanged)

Report negative results with the same care as wins. Every rejected experiment
gets its measured delta in the report. The headline benchmark may not regress
below the CI floor at any commit, the ladder must still end at
`ConsensusConfig::production()`, and any new benchmark must state its
denominator, its leakage story, and its dev/holdout numbers in the first
paragraph of its report. Write the full narrative to `IMPROVEMENT_REPORT_V9.md`.
